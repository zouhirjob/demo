**Diagramme de BPMN (Business Process Model and Notation) :**

![Diagramme de BPMN](https://i.imgur.com/JMbU7zL.png)

*Description du Diagramme de BPMN :*
1. **Processus Global : "Gérer les APIs"**
   - Démarre par l'authentification de l'utilisateur.
   - L'utilisateur sélectionne une API à gérer.
   - Le processus comprend la configuration de l'API, la validation des changements et l'application des modifications.

2. **Activités :**
   - **Identification et Authentification :**
     - L'utilisateur s'authentifie.
   - **Sélection de l'API :**
     - L'utilisateur choisit une API spécifique.
   - **Configuration :**
     - Inclut la modification des paramètres de l'API.
   - **Validation :**
     - Vérification des modifications avant application.
   - **Application des Modifications :**
     - Les modifications sont appliquées à l'API sélectionnée.

3. **Événements :
   - **Erreurs de Configuration :**
     - Si des erreurs surviennent pendant la configuration, l'utilisateur est informé et peut revenir en arrière pour corriger.
   - **Problèmes d'Authentification :**
     - Si l'authentification échoue, le processus est interrompu avec un message d'erreur.

4. **Passerelles (Gateways) :**
   - **Décision après Validation :**
     - Basée sur la validation, le processus peut bifurquer vers l'application des modifications ou le retour à la configuration.

**Remarques :**
- Les rectangles représentent les activités.
- Les losanges représentent les décisions ou points de contrôle.
- Les flèches indiquent la séquence des activités.
- Les cercles représentent les événements déclencheurs.

Ce schéma combine les notations UML et BPMN pour offrir une représentation complète et visuelle du processus de gestion des APIs avec Azure API Management.
